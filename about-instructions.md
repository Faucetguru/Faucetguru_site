# Faucet‑Guru Site – How It Works

## Overview
The site is a single‑page static application that displays a catalogue of crypto faucets, mining sites, PTC offers, and other reward‑type services.

* **HTML (`index.html`)** – defines the layout, loads the stylesheet and the two JavaScript files.
* **CSS (`style.css`)** – provides the dark‑theme UI, responsive grid, card hover effects and animations.
* **Data (`js/faucets.js`)** – a hard‑coded `window.faucetsData` array, each entry containing:
  - `id`, `name`, `type`
  - `trustScore`, `bonus`, `summary`
  - `referralLink`, `image`, `strategies`, `script`
  - `reviews[]` (user, rating, text)
* **Logic (`app.js`)** – renders the list, handles filtering, shows a detail view, and sanitises all dynamic content.
* **Validator (`tools/validate-faucets.js`)** – a Node script that checks the dataset for missing fields, dead or placeholder URLs, invalid trust scores, and reports a type summary.

## Main Features
1. **Dynamic Navigation** – Buttons are generated from the distinct `type` values found in the data, so any new type automatically appears as a filter.
2. **Safe Rendering** –
   - `escapeHtml()` escapes every text field before insertion into `innerHTML`.
   - `safeUrl()` forces only proper `http://`/`https://` URLs (blocks empty strings, `#`, and placeholder `TU_ID`).
   - External links include `rel="noopener noreferrer"`.
3. **Detail View** – Clicking a card hides the hero/list, shows full information, a larger screenshot, strategy, optional script snippet, and user reviews.
4. **Data Validation** – Run `node tools/validate-faucets.js` to see a summary, warnings (e.g., placeholder referral links) and any blocking errors.

## Usage Flow
1. Open `index.html` in a browser.
2. The navigation bar shows filters such as **Todas**, **Faucets**, **Minería**, **PTC**, plus any extra types present in the dataset.
3. Click a filter button to display only items of that type.
4. Click a card to open the detailed pane.
5. Use the back arrow (`← Volver al listado`) or the site logo to return to the list.

## Maintenance Tips
- **Add New Faucets** – Append a new object to `js/faucets.js`. The navigation will pick up the new `type` automatically.
- **Run the Validator** – `node tools/validate-faucets.js` before committing changes to catch missing fields or dead links.
- **Extend UI** – If you want additional sections (e.g., a real blog), replace the placeholder markup in `showBlog()`.
